### 4. **BasicData.cs** - 환급 시스템 추가

#### 새로운 필드
```csharp
protected Rank _rank;      // 아이템 등급
protected int _level = 1;  // 강화 레벨
```

#### 환급 시스템 메서드
```csharp
// 기본 환급 보상
public virtual (MoneyType moneyType, int amount) GetRefundReward()
{
    switch (_rank)
    {
        case Rank.Normal1:
        case Rank.Normal2:
            return (MoneyType.Gold, GetNormalRefundCoin());
        case Rank.Rare:
            return (MoneyType.Gold, GetRareRefundCoin());
        case Rank.Unique:
            return (MoneyType.Dia, 1);
        case Rank.Special:
            return (MoneyType.Dia, 2);
    }
}

// 레벨 기반 환급 (히든 레시피용)
public virtual (MoneyType moneyType, int amount) GetRefundRewardWithLevel(int level)
```

#### 등급별 환급 보상

| 등급 | 환급 보상 | 비고 |
|------|-----------|------|
| 노멀 | 코인 100 | 기본 값 (오버라이드 가능) |
| 레어 | 코인 500 | 기본 값 (오버라이드 가능) |
| 유니크 | 다이아 1 | 가챠 비용(10다이아) 대비 낮음 |
| 스페셜 | 다이아 2 | 가챠 비용(10다이아) 대비 낮음 |

## 📊 시스템 비교

### Before (강화 스크롤 시스템)
```
상점 레시피 → 코인으로 구매 → 강화 스크롤로 강화
히든 레시피 → 가챠 아이템 필요 → 강화 스크롤로 강화
```

### After (개편된 시스템)
```
상점 레시피 → 코인/다이아로 구매 → 코인/다이아로 강화 (Lv.1~3 코인, Lv.4~5 다이아)
히든 레시피 → 가챠로만 획득 → 중복으로 강화 (Lv.1→2: 1개, Lv.2→3: 2개...)
잉여 아이템 → 등급별 환급 (노멀/레어: 코인, 유니크: 다이아 1, 스페셜: 다이아 2)
```

## 🎮 게임플레이 시뮬레이션

### 상점 레시피 "김치찌개" 강화 예시
```
Lv.1: 구매 (코인 1000) → 판매가 500, 조리 10초
Lv.2: 강화 (코인 2000) → 판매가 1000, 조리 10초
Lv.3: 강화 (코인 4000) → 판매가 2000, 조리 10초
Lv.4: 강화 (다이아 10) → 판매가 4000, 조리 10초
Lv.5: 강화 (다이아 20) → 판매가 8000, 조리 10초
```

### 히든 레시피 "비밀 레시피" 강화 예시
```
Lv.1: 가챠 획득 → 판매가 5000, 조리 15초
Lv.2: 강화 (중복 1개) → 판매가 10000, 조리 15초
Lv.3: 강화 (중복 2개) → 판매가 20000, 조리 15초
...
Lv.6: 강화 (중복 5개) → 판매가 160000, 조리 13.5초 (-10%)
Lv.10: MAX → 판매가 2560000, 조리 7.5초 (-50%)
```

### 환급 시나리오
```
스페셜 등급 히든 레시피 Lv.5 판매
→ 다이아 2 × 5 = 다이아 10 환급
→ 가챠 비용(다이아 10)과 동일하게 회수
```
